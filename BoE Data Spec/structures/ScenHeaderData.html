Scenario Header Data

Size : 0xA3D6 bytes

 

Contains various informations and flags for the current scenario.

Data

Description

Scenario flags that specify on which platform the scenario has been made/saved. Actually used to guess the endianness of the scenario file.

Values

Platform/Presumed endianness flag1 flag2 flag3 flag4
Mac (Big Endian) 0x0a (10) 0x14 (20) 0x1e (30) 0x28 (40)
Windows (Little Endian) 0x14 (20) 0x28 (40) 0x3C (60) 0x50 (80)
If the game fails to find valid flags when reading a scenario header, it sets flag1 to 0.

Description

Various bits storing the scenario version, the minimum version of BoE to properly run it and the version of the Scenario Editor used to make it. That would allow some version control, however none has really been coded in the original source.

Values

  • ver[3] : version of the scenario in a x.y.z format.
  • min_run_ver : minimum version of the game to run the scenario. Set to 1 and never checked.
  • prog_make_ver[3] : version of the Scenario Editor in a x.y.z format. Always set to 1.0.0.
    The game prevent to load the scenario if x >= 2, which can never happen.

Description

Initialization values for various scenario variables (some needed to properly parse the scenario file).

  • num_towns : numbers of towns in the scenario.
  • out_width : width of the outdoor map.
  • out_height : height of the outdoor map.
  • difficulty : the general difficulty of the scenario (used by the difficulty adjustment algorithm).
  • intro_pic : the number of the picture appearing on the scenario selection dialog.
  • default_ground : unused (used to store the scenario rating on Win ?!?).

Values

  • num_towns : between 1 and 200 inclusive.
  • out_width : between 1 and 50 inclusive.
  • out_height : between 1 and 50 inclusive.
  • Additional condition : out_width * out_height <= 100
  • difficulty : between 0 (Low Level) and 3 (Very High Level).
  • intro_pic : between 0 and 29.
  • default_ground : set to 1 in Win code, unused on Mac.

Contains informations of the size and visibility of the scenario towns.

Description

Initialization values for various scenario variables (some needed to properly parse the scenario file).

  • town_size[200] : size of each town in the scenario (even for unused towns).
  • town_hidden[200] : visibility of each town in the scenario (even for unused towns).

Values

  • town_size[200] :
    Large town (64×64) Medium town(48×48) Small town(32×32)
    0 1 2
  • town_hidden[200] : Town visible is 0, town hidden is 1.

Seems totally unused and never set (0) ???

Flags used when checking the password of a scenario.

Values

  • flag_a : sizeof(scenario_data_type) + rand(-1000,1000) (unused in password calculation)
  • flag_b : town_s(password)
  • flag_c : out_s(password)
  • flag_d : init_data(password)
  • flag_e : str_size_1(password)
  • flag_f : str_size_2(password)
  • flag_g : 10 000 + rand(0,5000) (unused in password calculation)
  • flag_h : str_size_3(password)
  • flag_i : Never set (unused in password calculation)

Contains informations used in the initialization of the scenario.

Custom type used

Location

Description

  • intro_mess_pic : Not used (remnant of older games ?)
  • intro_mess_len : Not used (remnant of older games ?)
  • where_start : (x,y) Coordinates of the party start position in the initial town.
  • out_sec_start : (x,y) coordinates of the outdoor section in which the party starts.
  • out_start : (x,y) Coordinates of the party start position in the initial outdoor section.
  • which_town_start : number of the town the party start in.

Values

  • intro_mess_pic : 0
  • intro_mess_len : 0
  • where_start : each coordinate between 0 and scenario.town_size[which_town_start], inclusive.
  • out_sec_start : each coordinate between 0 and scenario.out_height/out_width, inclusive.
  • out_start : each coordinate between 0 and 48, inclusive.
  • which_town_start : between 0 and scenario.num_towns - 1.

Contains informations used in the initialization of the scenario.

Custom type used

Location

Description

  • town_data_size[200][5] : contains sizes of various town related data (town record, town terrain data, town strings data, talking strings data)
  • town_to_add_to[10] : the id (town_number) of up to 10 towns that checks for Party Stuff Done on entering and updates accordingly (aka Variable Town Entry).
  • flag_to_add_to_town[10][2] : the Party Stuff Done value to add (i.e add PSD[a][b] to town number).

Values

  • town_data_size[200][5] :
    town_data_size[X][0] town_data_size[X][1] town_data_size[X][2] town_data_size[X][3] town_data_size[X][4]
    size of town_record_type + size of big/ave/tiny_tr_type (depending on the town size) size of the 60 first town strings size of town strings between 60 and 159 size of talking_record_type + size of the first 80 talk strings size of talk strings between 80 and 169
  • town_to_add_to[10] : valid town_number (between 0 and scenario.num_towns-1).
  • flag_to_add_to_town[10][2] : valid PSD flags :
    • flag_to_add_to_town[X][0] : between 0 and 299
    • flag_to_add_to_town[X][1] : between 0 and 9

Size of outdoors related data.

Values

  • out_data_size[][0] : size of outdoor_record_type.
  • out_data_size[][1] : size of the 120 outdoors strings (out_strs[]).

Scenario town storage data.

Custom type used

RECT

Description

  • store_item_rects[3] : RECT in which items are saved upon leaving town.
  • store_item_towns[3] : up to 3 town ids (town_number) in which the item are saved on a given RECT.

Values

  • store_item_rects[3] : Any valid RECT (check against town boundaries, depending on town size).
  • store_item_towns[3] : valid town_number (between 0 and scenario.num_towns).

Scenario specials data.

Description

  • special_items[50] : described one of each of the 50 possible specials items.
  • special_item_special[50] : Which special to call when using a special item (if usable).

Values

  • special_items[50] :
    Value 0 1 10 11
    Meaning Non-usable item, party doesn't start with it. Usable item, party doesn't start with it. Non-usable item, party start with it. Usable item, party start with it.
  • special_item_special[50] : any valid special id (between 0 and 255 or -1 for none).

Various record type holding data for monsters, boats and horses.

Custom type used

Monster record type Boat record type Horse record type

Description

  • scen_monsters[256] : contains the data of the 256 defined monsters templates for the current scenario.
  • scen_boats[30] : contains the data of up to 30 boats (position, ownership, etc).
  • scen_horses[30] : contains the data of up to 30 horses (position, ownership, etc).

Scenario specials timers related data.

Description

  • scenario_timer_times[20] : any game time multiple of 10.
  • scenario_timer_specs[20] : which scenario special to fire up.

Values

  • scenario_timer_times[20] : the game time at which a scenario special is to fire up.
  • scenario_timer_specs[20] : Any integer between -1 (no special) and 255.
